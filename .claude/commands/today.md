Run the start-of-session routine.

## Steps

1. **Pull latest config** (if syncing to GitHub)
   - If this project has a GitHub sync configured, pull the latest config before anything else
   - Skip if no sync is configured

2. **Surface today's priorities**
   - Pull today's tasks from your task manager
   - List anything due today or overdue
   - Flag items that are blocking other work

3. **Review open commitments**
   - Scan memory for any commitments or deadlines that are coming up
   - Flag anything that needs attention before the session starts

4. **Set session context**
   - Ask what the user wants to accomplish this session
   - Note any time constraints

## Output format
Brief, scannable. Bullet points. Flag blockers clearly. Don't summarize things the user already knows — surface what needs attention.
